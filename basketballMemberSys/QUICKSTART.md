# 活動報名功能 - 快速開始指南

## 系統架構概述

該項目是一個篮球俱樂部會員管理系統，現已實現完整的**活動報名功能**。

```
┌─────────────────────────────────────────────────┐
│          前端 (Vue 3 + Element Plus)              │
├─────────────────────────────────────────────────┤
│  ├─ 活動報名頁面 (Activities.vue)                │
│  │   ├─ 活動卡片展示                             │
│  │   ├─ 按類型過濾                               │
│  │   └─ 報名對話框                               │
│  ├─ 管理員頁面 (AdminActivities.vue)             │
│  │   ├─ 活動列表表格                             │
│  │   ├─ 活動管理表單                             │
│  │   └─ 統計概覽                                 │
│  └─ 導航佈局 (MainLayout.vue)                    │
└──────────────────────────────────────────────────┘
                      ↓ (HTTP JWT)
┌──────────────────────────────────────────────────┐
│         後端 (Express.js + Prisma)                │
├──────────────────────────────────────────────────┤
│  ├─ 活動路由 (activityRoutes.js)                 │
│  │   ├─ 列表查詢                                 │
│  │   ├─ 詳情查詢                                 │
│  │   ├─ 報名(POST)                              │
│  │   ├─ 建立(POST-Admin)                         │
│  │   ├─ 編輯(PUT-Admin)                          │
│  │   └─ 刪除(DELETE-Admin)                       │
│  └─ 用戶路由 (userRoutes.js)                     │
│      └─ 孩子列表查詢                             │
└──────────────────────────────────────────────────┘
                      ↓ (ORM)
┌──────────────────────────────────────────────────┐
│    數據庫 (SQLite + Prisma Migration)            │
├──────────────────────────────────────────────────┤
│  ├─ Activity (活動表)                            │
│  ├─ Registration (報名表)                        │
│  ├─ User (用戶表)                                │
│  └─ Child (孩子表)                               │
└──────────────────────────────────────────────────┘
```

## 快速開始

### 前提條件
- Node.js 16+ 已安裝
- npm 或 yarn 包管理器

### 安裝步驟

#### 1. 後端設置

```bash
cd backend

# 安裝依賴
npm install

# 應用數據庫遷移（首次運行）
npm run prisma:migrate

# 啟動開發服務器
npm run dev
```

後端服務會運行在 `http://localhost:3000`

#### 2. 前端設置

```bash
cd frontend

# 安裝依賴
npm install

# 啟動開發服務器
npm run dev
```

前端應用會運行在 `http://localhost:5173`（或您終端顯示的端口）

### 驗證安裝

1. 打開瀏覽器訪問前端應用：`http://localhost:5173`
2. 使用以下測試帳號登入：
   - 用戶名：`testuser` 或 `admin`
   - 密碼：根據系統設置

3. 登入後應能看到導航欄中的「活動報名」選項

## 功能使用指南

### 用戶 - 活動報名

#### 步驟 1：瀏覽活動
1. 點擊導航欄「活動報名」
2. 查看活動卡片網格
3. 每個卡片顯示：
   - 活動名稱和類型標籤
   - 日期和時間
   - 地點和價格
   - 當前報名人數/最大人數
   - 進度條顯示報名情況

#### 步驟 2：過濾活動
- 使用頂部的標籤頁過濾：
  - 全部活動
  - 比賽
  - 訓練
  - 特別活動

#### 步驟 3：報名活動
1. 點擊希望報名的活動卡片上的「報名」按鈕
2. 在報名對話框中：
   - 選擇要報名的孩子（下拉列表）
   - 輸入緊急聯絡人名字
   - 輸入緊急聯絡電話
3. 點擊「報名」提交

#### 步驟 4：查看詳情
1. 點擊活動卡片上的「詳情」按鈕
2. 查看：
   - 完整活動介紹
   - 活動詳細資訊
   - 已報名参與者列表

### 管理員 - 活動管理

#### 步驟 1：訪問管理頁面
1. 以管理員身份登入（需要 ADMIN 角色）
2. 點擊導航欄「活動管理」

#### 步驟 2：查看活動列表
- 「活動列表」標籤顯示所有活動的表格
- 表格列包括：名稱、類型、日期、時間、地點、價格、人數進度

#### 步驟 3：建立新活動
1. 點擊「建立新活動」按鈕
2. 在表單中填入：
   - **活動名稱**：必填
   - **活動類型**：下拉選擇（比賽、訓練、特別活動）
   - **日期**：使用日期選擇器
   - **時間**：使用時間選擇器（格式 HH:MM）
   - **地點**：必填
   - **價錢**：可選（默認 0 = 免費）
   - **最大人數**：最少 1 人
   - **詳細資料**：可選的多行文本
3. 點擊「建立」提交

#### 步驟 4：編輯活動
1. 在活動列表中找到要編輯的活動
2. 點擊「編輯」按鈕
3. 修改相關字段
4. 點擊「更新」

#### 步驟 5：刪除活動
1. 在活動列表中找到要刪除的活動
2. 點擊「刪除」按鈕
3. 在確認對話框中確認刪除

> ⚠️ **警告**：刪除活動將同時刪除所有相關的報名記錄，此操作無法撤銷。

#### 步驟 6：查看統計概覽
1. 點擊「統計概覽」標籤
2. 查看統計卡片：
   - 總活動數
   - 各類型活動數量
3. 查看報名分佈表格

## API 端點參考

### 活動相關 (Authentication: JWT Bearer Token)

| 端點 | 方法 | 認證 | 角色 | 說明 |
|------|------|------|------|------|
| `/api/activities` | GET | ❌ | 任何 | 獲取所有活動 |
| `/api/activities/:id` | GET | ✅ | 任何 | 獲取活動詳情 |
| `/api/activities` | POST | ✅ | ADMIN | 建立活動 |
| `/api/activities/:id` | PUT | ✅ | ADMIN | 編輯活動 |
| `/api/activities/:id` | DELETE | ✅ | ADMIN | 刪除活動 |
| `/api/activities/:id/register` | POST | ✅ | 任何 | 報名活動 |

### 用戶相關

| 端點 | 方法 | 認證 | 說明 |
|------|------|------|------|
| `/api/user/profile` | GET | ✅ | 獲取個人檔案 |
| `/api/user/children` | GET | ✅ | 獲取孩子列表 |

## 常見問題排除

### 問題 1：無法連接到後端
**症狀**：前端提示連接錯誤

**解決方案**：
- 確保後端服務已啟動（`npm run dev` in backend 目錄）
- 檢查後端是否運行在 `http://localhost:3000`
- 檢查是否有防火牆阻止連接

### 問題 2：登入後看不到活動頁面
**症狀**：導航欄沒有「活動報名」選項

**解決方案**：
- 刷新頁面
- 檢查瀏覽器控制台是否有錯誤信息
- 確保已登入

### 問題 3：管理員看不到活動管理頁面
**症狀**：導航欄看不到「活動管理」選項

**解決方案**：
- 確保帳號角色為 ADMIN
- 嘗試登出再登入
- 檢查用戶角色設置

### 問題 4：報名失敗
**症狀**：點擊報名時出現錯誤

**可能原因**：
- 活動已滿額 ➜ 選擇未滿的活動
- 未選擇孩子 ➜ 在報名表單中選擇孩子
- 已報名相同活動 ➜ 檢查是否已報名

## 數據示例

### 建立活動 API 請求

```bash
curl -X POST http://localhost:3000/api/activities \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{
    "title": "友誼籃球賽",
    "type": "COMPETITION",
    "date": "2026-03-15",
    "time": "19:00",
    "location": "市體育中心館 1",
    "price": 50,
    "maxParticipants": 30,
    "description": "與隔壁鎮籃球隊的友誼比賽，歡迎所有年齡組參加。"
  }'
```

### 報名活動 API 請求

```bash
curl -X POST http://localhost:3000/api/activities/1/register \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{
    "childId": 1,
    "emergencyContact": "李媽媽",
    "emergencyPhone": "98765432"
  }'
```

## 性能優化建議

1. **圖片優化**：如後續添加活動圖片，使用適當的圖片格式和大小
2. **分頁**：當活動數量很多時，考慮添加活動列表分頁
3. **緩存**：前端可以緩存活動列表，減少 API 調用
4. **搜索功能**：添加活動搜索/排序功能

---

## 安全注意事項

1. **JWT 令牌**：存儲在 localStorage，生產環境應考慮使用 HttpOnly Cookie
2. **ADMIN 角色檢驗**：後端嚴格驗證管理員權限
3. **輸入驗證**：後端驗證所有輸入數據
4. **密碼**：不應在 URL 或請求體中傳輸密碼
5. **CORS**：生產環境應限制 CORS 源
